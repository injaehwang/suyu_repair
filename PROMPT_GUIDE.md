# 🛠 Project Guide: suyu repair (Frontend)

본 문서는 'suyu repair' 플랫폼의 프론트엔드 개발을 위한 **AI 에이전트 전용 지침서**입니다. 모든 UI/UX 개발 및 컴포넌트 설계 시 아래의 기술 스택과 비즈니스 로직을 엄격히 준수하십시오.

---

## 1. 개요 및 목표
* **서비스명:** suyu repair (비대면 의류 수선 플랫폼)
* **핵심 목표:** 사용자가 수선이 필요한 옷의 사진을 업로드하고, 견적 확인 후 결제 및 배송까지 원스톱으로 완료할 수 있는 심리스(Seamless)한 환경 제공.
* **타겟:** 모바일 환경에서 빠르고 간편하게 옷 수선을 맡기고 싶은 사용자.

## 2. 프론트엔드 기술 스택
* **Framework:** Next.js (App Router 권장)
* **Language:** TypeScript
* **Styling:** Tailwind CSS (Utility-first CSS)
* **State Management:** React Query (서버 상태 관리 및 캐싱)
* **Icons:** Lucide React (심플하고 모던한 아이콘 세트)

## 3. 핵심 UI/UX 가이드라인
### 3.1. 모바일 퍼스트 디자인
* 모든 화면은 모바일 브라우저 최적화를 최우선으로 하며, 데스크톱에서도 깨지지 않는 반응형 레이아웃을 유지합니다.
* 버튼은 엄지손가락으로 터치하기 쉬운 최소 크기(44x44px 이상)를 확보합니다.

### 3.2. 수선 문의 프로세스 (Step-by-Step)
1. **이미지 업로드:**
   - [앞면], [뒷면], [수선 부위 상세] 사진 3장을 필수로 요구합니다.
   - 업로드 전 클라이언트 측에서 이미지 압축을 진행하여 서버 부하를 줄입니다.
2. **수선 카테고리 선택:**
   - 기장 수선, 품 수선, 지퍼 교체, 단추/박음질 중 선택할 수 있는 칩(Chip) 형태의 UI를 제공합니다.
3. **요청서 상세:**
   - 텍스트 입력란(textarea)을 통해 구체적인 요구사항을 입력받습니다.

### 3.3. 실시간 상태 대시보드
- 사용자가 본인의 주문 상태를 한눈에 확인할 수 있는 스테퍼(Stepper) UI를 구현합니다.
- 상태값: `접수 완료` → `견적 산출` → `결제 대기` → `수선 중` → `배송 중` → `수선 완료`

## 4. 페이지 구성 요구사항
* **메인(Home):** 서비스 소개, 프로세스 안내, '수선 시작하기' CTA 버튼.
* **의뢰 페이지(Request):** 사진 업로드 및 정보 입력 폼.
* **내 주문(Order List):** 진행 중인 수선 내역 및 과거 이력 목록.
* **결제 및 확인(Payment):** 견적 금액 확인 및 외부 결제 모듈 연동 페이지.

## 5. 외부 서비스 연동 및 공통 컴포넌트
* **카카오톡 연동:** - 하단 고정(Fixed) 플로팅 버튼을 통해 1:1 상담 채널로 연결.
  - 알림톡 수신 동의 여부 체크박스 포함.
* **공통 Layout:** 상단 네비게이션 바(로고, 마이페이지 아이콘), 하단 푸터(CS 정보).
